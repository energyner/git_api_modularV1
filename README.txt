	git_api_modular

1. Description
git_api_modular is an innovative tool that simplifies advanced energy calculations through a robust backend and an intuitive frontend. Its accessibility allows for the automation of energy consumption processes and the assessment of their environmental impact, benefiting both industry experts and users seeking to optimize efficient energy use.
2. Purpose
This repository provides an accurate and efficient platform for complex energy calculations, reducing manual workload and boosting productivity. It also promotes technological solutions that facilitate decision-making regarding responsible energy use, contributing to environmental awareness and encouraging sustainable practices.
3. Goal
The objective is to develop an engaging and globally relevant tool that generates a significant impact on individual and collective participation. The goal is to inspire both users and developers to get involved in technological innovation and energy sustainability by promoting accessible and transformative solutions.
4. Project Structure
The project structure follows a well-defined organization based on the Git architecture, clearly separating the frontend and backend.
4.1. Frontend
• ./CLIENT/INDEX.HTML → User interface entry point.
• ./client/nav.mjs → Handles the communication logic with each app.
• ./client/style.css → Responsible for the visual presentation.
4.2. Backend
• ./SERVER/ENERGYSERV.MJS → Server core.
• ./calculations/energy-consumption.mjs → Energy consumption calculation.
• ./calculations/solar-production.mjs → Solar production estimation.
• ./calculations/footprint.mjs → Environmental impact and carbon footprint.
This modular structure facilitates project scalability and organization, allowing a clear separation between presentation logic and business logic.
________________________________________
5. Installation and Configuration
To use git_api_modular, ensure you meet the following requirements:
5.1. Prerequisites
• Knowledge of JavaScript, Node.js, and Express.
• Experience with PowerShell (PS) on Windows.
• Handling local and remote Git repositories.
• Node.js installed (https://nodejs.org/).
5.2. Installation Steps
5.2.1. Clone the repository:
a) git clone https://github.com/user/git_api_modular.git
b) cd git_api_modular
5.2.2. Install dependencies:
a) npm install
5.2.3. Run the server:
a) node ./SERVER/ENERGYSERV.MJS
5.2.4. Access the system:
b) Open your browser and visit http://localhost:3000 (if the configured port is 3000).
